## 🎤 DURING INTERVIEW: KEY REMINDERS

### General Demeanor
- [ ] Smile when you see the interviewer (genuine, not forced)
- [ ] Speak clearly (no mumbling, no rushing)
- [ ] Pause for 2-3 seconds before answering (shows you're thinking, not scripted)
- [ ] Make eye contact (look at camera, not screen)
- [ ] Hands visible (not hidden in lap)
- [ ] Lean slightly forward (shows engagement)
- [ ] Don't fidget with pen/phone

### If You Don't Know an Answer
✅ Do this:
- Pause and think (2-3 seconds)
- Say: "That's a good question. Let me think about that."
- Give your best attempt: "I'm not entirely sure, but I think..."
- Offer to check: "I could run this code to verify..."
- Admit uncertainty: "I haven't encountered that specific scenario..."

❌ Don't do this:
- Say "I don't know" immediately
- Guess confidently if unsure
- Try to sound smart with jargon you don't understand
- Ramble for 5+ minutes

### Bugs/Code Questions
- [ ] Read code carefully (don't rush)
- [ ] Explain out loud (even if you're unsure)
- [ ] Identify obvious bugs first, then subtle ones
- [ ] Trace through code logic step by step
- [ ] Explain WHY it's a bug (impact)
- [ ] Suggest fix and WHY it works
- [ ] Offer to run code if needed

### Project Questions
- [ ] Be specific (numbers, dataset names, exact architecture)
- [ ] Don't exaggerate results
- [ ] Admit limitations (every project has them)
- [ ] Show what you learned
- [ ] Connect to the role (how this prepares you for QUEST Lab)

### Paper Questions
- [ ] Don't pretend to understand if you don't
- [ ] Explain simply (not using fancy jargon unnecessarily)
- [ ] Give concrete examples
- [ ] Connect to your project if possible
- [ ] Show critical thinking (not just praise)

## 🚨 RED FLAGS (Things to Avoid)

### Communication Red Flags
- ❌ Speaking sounds robotic/scripted
- ❌ Can't explain your code in natural language
- ❌ Answers sound like ChatGPT (too formal/polished)
- ❌ Long pauses when asked follow-up questions
- ❌ Avoiding questions or being evasive

### Technical Red Flags
- ❌ Can't identify obvious bugs in code
- ❌ Suggesting fixes that don't actually work
- ❌ Not understanding your own project
- ❌ Wrong numbers (dataset size, metric values)
- ❌ Claiming methods you didn't actually use

### Interview Red Flags
- ❌ Others visible in background
- ❌ Phone camera positioned to hide workspace
- ❌ Suspicious pauses (like typing/reading)
- ❌ Noises indicating cheating (keyboard typing, voices)
- ❌ Late to call / technical issues not resolved

### Paper Red Flags
- ❌ Haven't actually read the paper
- ❌ Can't answer basic questions about methodology
- ❌ Generic comments (not specific)
- ❌ Criticism that shows you misunderstood paper
- ❌ No personal opinion (sounds rehearsed)

## 📊 SCORING RUBRIC (What They're Looking For)

They're assessing 5 things:

### 1. Technical Understanding (30%)
- [ ] Know your project deeply
- [ ] Can identify bugs in code

### 3. Authenticity (25%)
- [ ] Responses are your own (not ChatGPT)
- [ ] Body language/environment match
- [ ] Admit what you don't know
- [ ] Show genuine interest

### 4. Critical Thinking (15%)
- [ ] Paper review shows real reading
- [ ] Improvements are thoughtful (not generic)
- [ ] Can critique your own work
- [ ] Learn from failures

### 5. Cultural Fit (5%)
- [ ] Enthusiasm for deep learning
- [ ] Curiosity about research
- [ ] Growth mindset
- [ ] Can work independently

**You win if you're:**
- ✅ Technical AND clear (not just smart)
- ✅ Honest AND confident (not fake-confident)
- ✅ Specific AND thoughtful (not generic)
- ✅ Yourself AND professional (not robotic)
